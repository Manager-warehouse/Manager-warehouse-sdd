Xay dung he thong WMS Phuc Anh cho 3 kho vat ly (Hai Phong, Ha Noi, HCMC) voi cac nghiep vu core: nhap hang + QC, xuat hang + delivery, dieu chuyen noi bo, kiem ke, quan tri he thong, va ke toan cong no co ban. Backend Spring Boot 3.4.5 + Java 21, Frontend React 18 + Tailwind CSS. Day la Sprint 1 - tap trung vao nghiep vu kho cốt lõi.

## Technical Context

**Language/Version**: Java 21 (backend), JavaScript/React 18 (frontend)

**Primary Dependencies**:
- Backend: Spring Boot 3.4.5 (Web, JPA, Security, Validation, Flyway), Lombok, MapStruct, Jackson
- Frontend: React 18, React Router 6, Tailwind CSS 3.x, Axios, Zustand, React Hook Form, Lucide Icons
- Build: Maven (backend), Vite (frontend)

**Storage**: PostgreSQL 18 with Flyway migration; File system /uploads for attachments

**Testing**: JUnit 5 + Mockito (backend), Jest + React Testing Library (frontend)

**Target Platform**: Linux server / Docker Compose (Dev), web browser (Chrome, Firefox)

**Project Type**: Full-stack web application + REST API

**Performance Goals**: 1000+ transactions/month, < 2s page load, < 500ms API response time (p95)

**Constraints**: 
- Inventory quantity NEVER negative (DB constraint + app-level validation)
- Optimistic locking on all inventory updates
- Audit log for all warehouse operations
- FEFO/FIFO batch selection mandatory
- RBAC with both role AND warehouse scope
- Max 300 lines/file, 40 lines/function in Java

**Scale/Scope**: 3 warehouses, 1000+ products, 50+ dealers, 1000+ transactions/month

### Gates from Constitution:

1. **GATE-01 Layered Architecture**: Must follow Controller -> Service -> Repository -> Entity.
   - Status: PASS - standard Spring Boot layout

2. **GATE-02 Inventory Integrity**: quantity >= 0, @Version optimistic locking, audit log.
   - Status: PASS - enforced by domain rules

3. **GATE-03 FEFO/FIFO**: Expiry-based and date-based batch selection.
   - Status: PASS - explicit business requirement

4. **GATE-04 QC Gate**: Failed QC -> Quarantine, excluded from available.
   - Status: PASS - ADR-005 implemented

5. **GATE-05 In-Transit**: Transfer thru virtual warehouse.
   - Status: PASS - ADR-004 implemented

6. **GATE-06 Auth & RBAC**: JWT + bcrypt + role + warehouse scope.
   - Status: PASS - standard Spring Security

7. **GATE-07 Test Coverage**: >= 80% service coverage.
   - Status: PASS - enforced by Definition of Done

8. **GATE-08 No raw SQL**: All DB access thru JPA/Hibernate.
   - Status: PASS - prohibited by constitution

9. **GATE-09 No negative inventory**: Constraint at DB + app level.
   - Status: PASS - lesson from LESSON-001

10. **GATE-10 Batch single grade**: Moi batch chi 1 grade.
    - Status: PASS - lesson from LESSON-002
