# Data Model: Planner Receipt Drafting

## Receipt

Represents a supplier purchase receipt document created before physical receiving.

**Fields**

| Field | Type | Rules |
|-------|------|-------|
| `id` | Long | Generated primary key |
| `receipt_number` | String | Required, unique, generated by system |
| `source_order_code` | String | Required source reference / PO number |
| `type` | Enum | Must be `PURCHASE` in this flow |
| `warehouse_id` | Long | Required, active warehouse, actor must be authorized |
| `supplier_id` | Long | Required, active supplier |
| `dealer_id` | Long | Must be null in this flow |
| `contact_person` | String | Required, non-blank |
| `source_channel` | Enum/String | Required, only `ZALO` or `EMAIL` |
| `status` | Enum | Created as `PENDING_RECEIPT` |
| `document_date` | LocalDate | Required, defaults to request date or current business date |
| `created_by` | Long | Required actor |
| `notes` | String | Optional |
| `created_at` | OffsetDateTime | System timestamp |
| `updated_at` | OffsetDateTime | System timestamp |

**Validation**

- Reject `RETURN` or dealer return inputs in this endpoint.
- Reject duplicate `source_order_code` for the same supplier and warehouse purchase receipt scope.
- Reject inactive supplier or warehouse.
- Reject unauthorized warehouse access.

**State Transition**

```text
CREATE -> PENDING_RECEIPT
```

No inventory, batch, QC, quarantine, or approval transition occurs in this feature.

## ReceiptItem

Represents one expected product line on the drafted receipt.

**Fields**

| Field | Type | Rules |
|-------|------|-------|
| `id` | Long | Generated primary key |
| `receipt_id` | Long | Required parent receipt |
| `product_id` | Long | Required, active product |
| `expected_qty` | Integer | Required, integer greater than zero |
| `actual_qty` | BigDecimal | Null at drafting |
| `over_received_qty` | BigDecimal | Defaults to 0 |
| `batch_id` | Long | Null at drafting |
| `location_id` | Long | Null at drafting |
| `qc_*` fields | Various | Null at drafting |

**Validation**

- Each item must contain exactly one `product_id` and one `expected_qty`.
- `expected_qty` must be positive integer input.
- Product must be active.
- Duplicate product lines are allowed only if the implementation has a clear later lot-splitting reason; otherwise reject duplicates in the request to keep one expected line per product.

## Request DTO

`CreateReceiptRequest`

| Field | Type | Rules |
|-------|------|-------|
| `supplier_id` | Long | Required |
| `contact_person` | String | Required, non-blank |
| `warehouse_id` | Long | Required |
| `source_reference` | String | Required, PO number |
| `source_channel` | Enum | Required, `ZALO` or `EMAIL` |
| `items` | List | Required, at least one item |
| `notes` | String | Optional |

`CreateReceiptItemRequest`

| Field | Type | Rules |
|-------|------|-------|
| `product_id` | Long | Required |
| `expected_qty` | Integer | Required, positive integer |

## Response DTO

`ReceiptResponse`

Return created receipt id, generated receipt number, source reference, supplier id, warehouse id, source channel, status, document date, item summaries, created by, and timestamps.

## Repository Needs

- `ReceiptRepository.existsBySupplierIdAndWarehouseIdAndSourceOrderCodeAndType(...)` or equivalent derived query.
- Product, Supplier, and Warehouse repository lookups that include active-state validation.
- Receipt item persistence through a repository or cascade from `Receipt`.
- `DocumentSequenceRepository.findBySequenceKeyForUpdate("RECEIPT")` to allocate race-safe receipt numbers without raw SQL.

## DocumentSequence

Stores document counters used for deterministic document number allocation.

| Field | Type | Rules |
|-------|------|-------|
| `sequence_key` | String | Primary key, `RECEIPT` for receipt numbers |
| `next_value` | Long | Required, positive, incremented under pessimistic write lock |
| `updated_at` | OffsetDateTime | Updated whenever the counter is allocated |

## Audit Data

`before`: null

`after`: receipt number, type, status, supplier id, warehouse id, source reference, source channel, item count, item product ids and expected quantities.
